# Fast Local Gates

## Goal

Keep the permanent requirement-architecture contract complete while making repeated local execution reuse compiled artifacts and removing architecture-only CI.

## Approach

1. Give each Specular Cargo test a stable ignored target directory under `.cargo-target/gate/targets/` instead of a deleted temporary target. Detached pre-push worktrees reuse repository-owned gate and Fixture3 cache roots with repository-relative workspace identities.
2. Keep the AQC local gate responsible for locked Cargo metadata, cargo-deny, strict Clippy where configured, tests for every production package and tool workspace, every Fixture3 suite, the permanent Specular contract, and the AQC self-scan. Tool workspaces without a local Clippy file use the repository configuration.
3. Delete the architecture-only GitHub workflow and replace CI promises in the plan, spec, coverage, and conventions with local-gate requirements.
   Wire the AQC local gate through the repository-owned pre-push hook. Keep pre-commit change-scoped.
4. Fix the final checker counterexamples:
   - reject local and cross-crate helper-produced membership;
   - track type-annotated membership locals;
   - verify `ItemRequirements` and `KeyedItem` originate from `aqc-file-engine-core` or a proven public re-export chain;
   - stop classifying unrelated `*_keys` fields as membership by name.
   - require the canonical core manifest so a counterfeit package name cannot establish core vocabulary;
   - clear tracked provenance when a local binding shadows an earlier membership transfer.
   - resolve imports, chained parent aliases, qualified roots, and local same-name shadows by module identity;
   - discover `#[path]` modules nested inside inline modules;
   - preserve inferred closure identity through destructuring and reassignment.
   - reject glob and block-local imports in requirement crates because they make the static requirement surface scope-dependent;
   - inventory imported semantic type aliases and distinguish generic or unrelated same-terminal types from requirement roots.
5. Add one executable adversarial fixture per counterexample and require each test by exact name through Specular.
6. Fix the strict-Clippy failure in the Deny test helper without changing production behavior.
   Fix JSON and YAML strict-Clippy failures exposed after intermediate command status propagation became effective, without changing parsing behavior.
   Record the standardized source-shaped waiver for Deny's exhaustive field-to-field merge mapping rather than splitting one schema inventory across partial modules.
7. Generate Cargo patch configuration with atomic replacement so concurrent local checks cannot read partial files.
   Do not replace an unchanged generated configuration, because changing its metadata invalidates Cargo fingerprints.
8. Keep fixture Cargo homes and targets under the clone-local `.cargo-target/fixtures/` tree so independent clones cannot collide.
9. Bind pre-push verification to immutable pushed bytes: every non-deletion ref must equal checked-out `HEAD`, the repository must be clean, and the gate runs in a detached worktree at that commit while reusing the repository cache.
10. Keep the existing change-scoped G3RS source checks, then validate repository and workspace configuration from the staged-index worktree generated by the Shakrs hook policy.
    Dispatch both registered pre-commit fragments through the executable syntax recognized by the repository's modular hook contract.
    Supply each staged G3RS workspace check with generated patches for only its transitive local dependencies, so independently published workspaces resolve without unused patches changing locked metadata.
11. Run a fast failure-injection check for metadata, cargo-deny, Clippy, and test so parallel workers cannot hide an earlier failed stage.
12. Give Specular and the workspace gate the same generated Cargo home and target identity so one gate phase cannot invalidate the other's Cargo fingerprints.

## Decisions

- The permanent Specular contract remains the sole owner of architecture-checker execution.
- Caches are ignored build products and may persist across local runs.
- Source identity, not terminal type spelling, determines canonical core vocabulary.
- Core requirement vocabulary keeps its established names; renamed imports are rejected.
- Adapter membership output accepts only visible policy-field transfer, visible destructuring, or `ItemRequirements::map`.
- No compatibility aliases or old workflow files remain.
- The release workflow remains release orchestration; ordinary architecture and behavior checks run locally.

## Acceptance

- Every new adversarial fixture produces its own expected violation or accepted result.
- Every AQC production workspace passes locked metadata, cargo-deny, configured strict Clippy, and tests.
- Every AQC Fixture3 suite passes.
- Checker tests, cargo-deny, and strict Clippy pass.
- Both repository scans report no violation.
- A first and second permanent Specular verification pass; the second reuses Cargo artifacts.
- A final adversarial review finds no surviving counterexample or stale CI contract.
- Pre-push verifies the exact pushed `HEAD` from a clean repository.
- Fixture Cargo state is isolated by clone under `.cargo-target/fixtures/`.

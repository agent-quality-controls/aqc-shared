## 3. Pattern Verdict Matrix

- Consequence: adapters can build engine requirements without depending on core directly

- Pattern: `src/engine.rs` implements `FileEngine<Resolved*>` and erased `Engine` through `merged_reconcile`
- Source module: all current AQC TOML engines
- Verdict: REUSE
- Consequence: deny participates in the same runner/adapter engine pipeline
- Status: exact accepted contract

Row PATTERN-003

- Pattern: fixed public requirement fields for small fixed file structures
- Source module: `aqc-rust-toolchain-toml-engine`
- Verdict: REUSE
- Consequence: deny field names model `deny.toml`, not policy names or generic maps
- Status: exact accepted contract

Row PATTERN-004

- Pattern: generic `BTreeMap<String, ConfigScalar>` for known settings
- Source module: older setting-map designs
- Verdict: REJECT
- Consequence if copied: invalid enum values and wrong scalar families become runtime strings instead of typed requirements
- Status: exact rejected design

Row PATTERN-005

- Pattern: domain-specific item types implement `FileItemRequirement`
- Source module: Cargo dependencies, Clippy disallowed entries
- Verdict: REUSE
- Consequence: core handles required/forbidden/closed merging while deny owns file-specific identity and composition
- Status: exact accepted contract

Row PATTERN-006

- Pattern: public composite structs with public fields
- Source module: older AQC file values
- Verdict: REJECT

- Status: exact rejected design

Row PATTERN-007

- Pattern: local TOML array item reconciliation inside each engine
- Source module: Clippy disallowed arrays and Cargo table items
- Verdict: REJECT for generic array-backed item mechanics; REUSE only domain-specific item parsing, rendering, and glob-specific logic

- Pattern: Clippy disallowed path-glob reconciliation
- Source module: `aqc-clippy-toml-engine`
- Verdict: KEEP LOCAL
- Consequence: TOML core owns item mechanics; Clippy still owns glob compilation and glob-vs-path findings
- Status: exact accepted contract

## 8. TOML Core Item Helper Contract

Add `packages/file-types/toml/aqc-toml-engine-core/src/items/`.

Public helper surface:

```rust
pub trait TomlArrayItem: FileItemRequirement {
    fn read_value(value: &toml_edit::Value) -> Result<Self, String>;
    fn write_value(&self) -> toml_edit::Value;
    fn matches_value(current: &Self, required: &Self) -> bool;
    fn render_value(&self) -> String;
}

pub trait TomlArrayTableItem: FileItemRequirement {
    fn read_table(table: &dyn toml_edit::TableLike) -> Result<Self, String>;
    fn write_table(&self) -> toml_edit::Table;
    fn matches_table(current: &Self, required: &Self) -> bool;
    fn render_table(&self) -> String;
}

pub struct TomlItemField<'a> {
    pub table_path: &'a [&'a str],
    pub field_key: &'a str,
    pub display_key: &'a str,
}

pub fn reconcile_array_items<Item>(
    doc: &mut toml_edit::DocumentMut,
    field: TomlItemField<'_>,
    requirements: &ResolvedItemRequirements<Item>,
    findings: &mut Vec<Finding>,
)
where
    Item: TomlArrayItem,
    Item::Identity: ToString;

pub fn reconcile_array_table_items<Item>(
    doc: &mut toml_edit::DocumentMut,
    field: TomlItemField<'_>,
    requirements: &ResolvedItemRequirements<Item>,
    findings: &mut Vec<Finding>,
)
where
    Item: TomlArrayTableItem,
    Item::Identity: ToString;
```

Helper behavior:

- required missing item writes one item
- required present-but-different item replaces the item
- forbidden item removes every matching identity
- closed collection removes extras not in required identities
- malformed array shape reports a mismatch and does not silently drop data
- duplicate identity reports a mismatch and does not write a replacement for that collection
- item parsing stays in the concrete engine through trait methods

Existing consumer migration:

- `aqc-clippy-toml-engine/src/reconcile/disallowed.rs` uses `reconcile_array_items`
- `DisallowedEntry` implements `TomlArrayItem` for the Clippy file shape:
  - string item: `"clippy::some_lint"` means path only
  - inline table item: `{ path = "clippy::some_lint", reason = "..." }` means path plus reason
- Clippy keeps `apply_forbidden_path_globs` local because forbidden globs are Clippy semantics, not TOML item mechanics
- Clippy removes local `apply_required`, `apply_forbidden`, `prune_extras`, and malformed-array helpers that duplicate TOML core

Status: exact accepted contract

## 9. Validation And Init Behavior

- invalid UTF-8
- malformed TOML
- wrong scalar type
- wrong list member type
- empty list member
- wrong item shape
- unknown closed enum value
- duplicate item identity inside one array-backed collection
- feature entry with overlapping allow and deny feature names

Repairable drift:

- missing file
- missing managed section
- missing managed field
- scalar value mismatch
- list contains/excludes/exact mismatch
- valid extra entry in a closed collection
- deprecated package-spec spelling `name` repaired to canonical `crate` where cargo-deny accepts both

Valid pass-through:

- valid `[bans.build]` fields when no requirement forbids them
- valid `[advisories].version` when no requirement forbids it
- valid `[licenses].version` when no requirement forbids it
- valid unknown sections only when `closed_settings` is absent

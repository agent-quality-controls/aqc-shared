# AQC Shared 0.2.0 Release

## Goal

Push the current `aqc-shared` main branch and publish the workspace crates as
`0.2.0`.

## Approach

- Bump the workspace package version from `0.1.0` to `0.2.0`.
- Update internal published dependencies on `aqc-file-engine-core` from
  `0.1.0` to `0.2.0`.
- Run the workspace validation and publish dry-run before pushing.
- Push `main`.
- Publish crates in dependency order: core first, then independent packages and
  TOML engines.

## Decisions

- Use `0.2.0` instead of `0.1.1` because the commits include breaking public
  API changes such as making `FileTree` fields private.
- Publish the whole workspace version consistently because all crates inherit
  the workspace version.

## Files To Modify

- `Cargo.toml`
- `packages/file-types/toml/aqc-cargo-toml-engine/Cargo.toml`
- `packages/file-types/toml/aqc-clippy-toml-engine/Cargo.toml`
- `packages/file-types/toml/aqc-rustfmt-toml-engine/Cargo.toml`
- `Cargo.lock`

## Verification

- `cargo update --workspace`
- `cargo publish --dry-run --workspace --allow-dirty`
- `g3rs validate workspace --path .`

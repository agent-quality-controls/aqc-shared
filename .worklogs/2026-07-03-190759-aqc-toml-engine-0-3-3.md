# AQC TOML Engine 0.3.3 Release

## Summary

- Bumped Cargo, Clippy, and Rustfmt TOML engine crates from 0.3.2 to 0.3.3.
- This release is needed because the published 0.3.2 crates do not expose the file-engine-core re-export surface required by Shakrs adapters.

## Decisions

- Published a patch release instead of changing adapters to import aqc-file-engine-core directly.
- Reason: adapters must consume the exact engine-supported assertion surface through the engine crate boundary.
- Cargo.lock changes only update the package self-version entries.

## Key Files

- packages/file-types/toml/aqc-cargo-toml-engine/Cargo.toml
- packages/file-types/toml/aqc-clippy-toml-engine/Cargo.toml
- packages/file-types/toml/aqc-rustfmt-toml-engine/Cargo.toml

## Next Steps

- Publish the three 0.3.3 engine crates.
- Point Shakrs adapter and CLI manifests at 0.3.3.

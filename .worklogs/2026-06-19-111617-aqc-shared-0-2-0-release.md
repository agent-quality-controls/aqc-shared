# AQC Shared 0.2.0 Release

## Summary

Prepared the workspace for a `0.2.0` crates.io release. The release is a
minor pre-1.0 bump because the current commit set includes public API changes,
including the `FileTree` accessor change and Cargo/core requirement API work.

## Decisions Made

- Used `0.2.0` instead of `0.1.1` because pre-1.0 breaking public API changes
  should move the minor version.
- Kept all workspace crates on the same version because package versions
  inherit from the workspace.
- Updated TOML engine dependencies on `aqc-file-engine-core` to `0.2.0` so
  published engine crates use the matching core API.

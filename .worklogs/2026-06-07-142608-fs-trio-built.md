# aqc-fs-utils + aqc-filetree + aqc-git-helpers built

## Summary
Built the three planned-but-empty shared packages (each had a settled
`plan.md`; needed soon by spec3). Manifest-first: `plan.md.manifest.toml`
written beside each plan (single audited pass), then implemented to the
manifests.

## What landed
- **aqc-fs-utils**: `read_text` / `read_bytes` with the fixed rules ‚Äî strict
  UTF-8, raw-NUL rejection before decode, 1 GiB default cap enforced at BOTH
  stat and read time (TOCTOU), `SymlinkReadPolicy`, opt-in CRLF
  normalization. All failures are `Err`; no "binary/skip" Ok branch.
- **aqc-filetree**: `build_file_tree` two-phase walk on the `ignore` crate
  (ripgrep's walker): phase 1 gitignore-aware; phase 2 (opt-in
  `RecoveryRules`) re-walks WITHOUT gitignore and recovers matching paths as
  `EntryOrigin::Recovered` ‚Äî `skip_dir_names` prunes descent in both phases
  (the plan's reason for having both mechanisms). `FileTree`
  entry/entries_with_origin/glob queries (`globset`), `SkipDirPreset`
  constants, `SymlinkPolicy` Skip|Record|Follow, sorted `rel_path` entries.
- **aqc-git-helpers**: `git status --porcelain=v1 -z` subprocess + parser
  (`parse_porcelain_v1z` is public ‚Äî testable on fixture strings, no repo);
  `worktree_changes` / `is_worktree_clean` / `changes_affecting_paths` /
  `dirty_paths`. Extraction gap resolved in the manifest: the path filter is
  a DIRECTORY BOUNDARY (`x` or `x/...`), never substring ‚Äî pinned by a test
  (`specs` does not match `specs-other/`). Renames match via `old_path` too.

## Decisions
- **Per-crate `clippy.toml` capability overrides**: the workspace-wide
  std::fs / Command::new bans exist to force code through centralized I/O
  layers ‚Äî these crates ARE those layers, so each carries a copy of the
  workspace clippy.toml with exactly its own capability un-banned (fs reads
  for fs-utils/filetree, Command::new for git-helpers, fs writes for their
  test fixtures). Everything else stays workspace-identical.
- `WalkError` variants (plan named the type only): RootNotFound /
  RootNotADirectory / Io ‚Äî recorded in the manifest.
- New deps `ignore` + `globset` (ripgrep family, maintained) added to the
  repo allowed_deps.

## Verification
Workspace: clippy 0, all tests (incl. 8 walk + 8 read + 7 porcelain), fmt,
dupes 7.0% (pass). Per-manifest verification commands
(`cargo test -p <crate>`) green.

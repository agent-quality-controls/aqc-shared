# TOML Engines 0.7

## Summary

Coordinated all five concrete TOML engines with file-engine-core and TOML-engine-core 0.7. Existing reconciliation behavior is preserved while findings now carry the shared optional selector field.

## Decisions Made

- Updated every concrete TOML engine in one generation so no engine retains the old finding or core contract.
- Added `selector: None` only where the file finding has no item selector; glob-capable engines continue to attach selectors where available.
- Regenerated locks using only the two approved local core patches so source verification remains locked before registry release.
- Added the strict `std::process::abort` Clippy baseline to the affected engines.

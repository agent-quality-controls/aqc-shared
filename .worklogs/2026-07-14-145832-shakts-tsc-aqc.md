# Shakts TSC AQC

## Summary

Added reusable JSONC mechanics, a TSConfig engine, and Package JSON script/dev-dependency requirements for the Shakts TSC vertical. Unified resolved-requirement attribution across existing format engines without adding a second finding-construction API.

## Decisions Made

- Keep syntax selection in concrete engines through `JsoncParseOptions`; JSONC core owns parsing and editing only.
- Keep `Finding::Mismatch` construction in format and concrete engines; core exposes only universal attribution projection.
- Model Package JSON scripts and development dependencies as keyed scalar maps using existing core assertions.
- Reject duplicate JSONC keys recursively and preserve unchanged bytes, comments, trailing commas, BOM, extended numeric literals, and unmanaged values during replacement and insertion.
- Use Tree-sitter only to identify TypeScript compiler-mode numeric literals before lossless JSONC CST reconciliation; the concrete TSConfig engine opts into those syntax extensions.
- Use exact transitive local patches for the AQC fixture probe so unrelated workspaces cannot invalidate its lock.
- Share the transitive fixture Cargo-home builder between PNPM and TSC AQC probes.

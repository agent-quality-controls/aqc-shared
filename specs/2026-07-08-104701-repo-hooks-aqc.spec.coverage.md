# Repo Hooks AQC Spec Coverage

## AQC File Engine Core

- Covered by content checks for multi-file `EngineFileState`, `EngineFileOutput`, `target_paths`, and `expected_executable`.
- Covered by forbidden content checks for tool/product names and `ExecutableRequirement`.

## Text Engine Core

- Covered by tree checks for package, model, merge, reconcile, and tests.
- Covered by content checks for `ItemRequirements<TextFileRequirement>`, `ItemRequirements<TextSnippet>`, `ResolvedItemRequirements`, scalar exact contents, and executable scalar.
- Covered by forbidden checks for `EngineRequirement` and the old `aqc-text-file-engine` package name.
- Covered by custom `text-core-contract`.

## Git Hooks Engine

- Covered by tree checks for package, model, merge, and tests.
- Covered by content checks for `GitHooksPath`, `GitHooksValueError`, `GitHooksRequirements`, and `ResolvedGitHooksRequirements`.
- Covered by dependency checks for AQC-only dependencies and no dependency on another file-engine crate.

## Dependency Boundaries

- Covered by dependency checks forbidding Shackles and Shakrs crates from AQC engines.
- Covered by content checks forbidding product, policy, adapter, and command execution names.

## UNCOVERED

- None.
